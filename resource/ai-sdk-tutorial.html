CHAPTER 01
什么是 AI SDK？

Vercel AI SDK（npm 包名 ai）是目前最流行的 JavaScript/TypeScript AI 集成库，专为 Next.js、Node.js、React 等场景设计。

🎯 核心价值
一套统一的 API，支持 OpenAI、Anthropic Claude、Google Gemini、Mistral 等几乎所有主流模型提供商。切换模型只需换一行代码。
核心包
ai @ai-sdk/openai @ai-sdk/anthropic @ai-sdk/google

ai 是核心库，provider 包按需安装。

四大核心功能
generateText streamText generateObject streamObject

覆盖 99% 的 AI 应用场景。

与直接调用 OpenAI SDK 的区别

❌ 直接用 OpenAI SDK

// 每个模型写法不同，难以切换
const res = await openai.chat
  .completions.create({
    model: 'gpt-4',
    messages: [...]
  })
const text = res.choices[0]
  .message.content

✅ 用 AI SDK

// 统一 API，模型随便换
const { text } = await generateText({
  model: openai('gpt-4'),
  prompt: '...'
})

CHAPTER 02
安装与配置

以 Next.js + OpenAI 为例，演示完整的初始化流程。其他 provider 逻辑相同。

Step 1 — 创建 Next.js 项目
npx create-next-app@latest my-ai-app --typescript
cd my-ai-app
Step 2 — 安装依赖
npm install ai @ai-sdk/openai

# 如果用 Anthropic Claude：
npm install ai @ai-sdk/anthropic

# 如果用 Google Gemini：
npm install ai @ai-sdk/google
Step 3 — 配置 API Key（.env.local）
# OpenAI
OPENAI_API_KEY=sk-xxxxxxxxxxxxxxxx

# Anthropic
ANTHROPIC_API_KEY=sk-ant-xxxxxxxxxxxxxxxx

# Google
GOOGLE_GENERATIVE_AI_API_KEY=AIzaxxxxxxxx
⚠️ 永远不要把 .env.local 提交到 Git！确认 .gitignore 中已包含它。API key 的 provider 会根据包名自动读取对应的环境变量。
Provider 初始化方式对比
import { openai } from '@ai-sdk/openai'
import { anthropic } from '@ai-sdk/anthropic'
import { google } from '@ai-sdk/google'

// 用法完全一样，只换 model 对象！
const model1 = openai('gpt-4o')
const model2 = anthropic('claude-3-5-sonnet-20241022')
const model3 = google('gemini-1.5-pro')
💡 AI SDK 会自动从环境变量读取 API Key。比如 @ai-sdk/openai 会自动读取 OPENAI_API_KEY，你不需要手动传入。
CHAPTER 03
generateText — 文本生成

最基础的功能：发一个 prompt，得到一个完整的文本回复。适合不需要实时显示的场景（分类、摘要、翻译等）。

最简单的例子
import { generateText } from 'ai'
import { openai } from '@ai-sdk/openai'

const { text } = await generateText({
  model: openai('gpt-4o-mini'),
  prompt: '用一句话解释什么是量子纠缠',
})

console.log(text)
// 量子纠缠是指两个粒子的状态相互关联...
带 system prompt 的对话
const { text } = await generateText({
  model: openai('gpt-4o-mini'),
  system: '你是一个专业的代码审查工程师，用简洁的中文回答',
  prompt: '这段代码有什么问题：for(var i=0;i<10;i++){}',
})

console.log(text)
多轮对话（messages 格式）
const { text } = await generateText({
  model: openai('gpt-4o-mini'),
  messages: [
    { role: 'user', content: '我叫小明' },
    { role: 'assistant', content: '你好，小明！有什么我能帮你的？' },
    { role: 'user', content: '你还记得我叫什么吗？' },
  ],
})

console.log(text)
// "当然记得，你叫小明！"
返回值解构 — 不止有 text
const result = await generateText({
  model: openai('gpt-4o-mini'),
  prompt: '讲个冷笑话',
})

console.log(result.text)           // 主要内容
console.log(result.usage)          // { promptTokens, completionTokens, totalTokens }
console.log(result.finishReason)   // 'stop' | 'length' | 'tool-calls'
console.log(result.response)       // 原始响应对象
在 Next.js API Route 中使用
// app/api/summarize/route.ts
import { generateText } from 'ai'
import { openai } from '@ai-sdk/openai'
import { NextRequest } from 'next/server'

export async function POST(req: NextRequest) {
  const { content } = await req.json()
  
  const { text } = await generateText({
    model: openai('gpt-4o-mini'),
    system: '将用户输入摘要为3句话',
    prompt: content,
  })
  
  return Response.json({ summary: text })
}

CHAPTER 04
streamText — 流式输出

让 AI 的回答像 ChatGPT 一样"一个字一个字"地出现。这是现代 AI 应用最核心的体验。

🌊 流式输出原理：AI 模型每生成一个 token（词片段）就立即发送，而不是等全部生成完再返回。这让用户感知到响应时间极短。
Server Side — Next.js API Route
// app/api/chat/route.ts
import { streamText } from 'ai'
import { openai } from '@ai-sdk/openai'

export async function POST(req: Request) {
  const { messages } = await req.json()

  const result = streamText({
    model: openai('gpt-4o-mini'),
    messages,
  })

  // 关键：把流转成 Response 返回给前端
  return result.toDataStreamResponse()
}
Client Side — useChat Hook（最简方式）
// app/page.tsx
'use client'
import { useChat } from 'ai/react'

export default function Chat() {
  const { messages, input, handleInputChange, handleSubmit } = useChat()

  return (
    <div>
      {messages.map(m => (
        <div key={m.id}>
          <b>{m.role}</b>: {m.content}
        </div>
      ))}
      <form onSubmit={handleSubmit}>
        <input value={input} onChange={handleInputChange} />
        <button type="submit">发送</button>
      </form>
    </div>
  )
}

// 🎉 就这些！已经是一个可用的流式聊天界面了
useChat 的完整能力
const {
  messages,         // 聊天记录数组
  input,            // 当前输入框的值
  handleInputChange,// 输入框 onChange 处理器
  handleSubmit,     // 表单 onSubmit 处理器
  isLoading,        // 是否正在等待 AI 回复
  stop,             // 中断当前生成
  reload,           // 重新生成最后一条回复
  error,            // 错误信息
} = useChat({
  api: '/api/chat',  // 默认值，可自定义
  initialMessages: [],// 初始消息
  onFinish: (msg) => { /* 生成完成回调 */ },
  onError: (err) => { /* 错误回调 */ },
})
💡 useChat 是 AI SDK 最常用的 Hook，它自动管理消息历史、处理流式数据解析，以及状态更新。90% 的聊天应用用它就够了。
不用 Hook，手动处理流（Node.js 环境）
import { streamText } from 'ai'
import { openai } from '@ai-sdk/openai'

const result = streamText({
  model: openai('gpt-4o-mini'),
  prompt: '写一首五言绝句',
})

// 逐 chunk 处理
for await (const chunk of result.textStream) {
  process.stdout.write(chunk) // 实时打印每个 token
}

// 或者等待全部完成
const fullText = await result.text
CHAPTER 05
Tool Use — 工具调用

让 AI 不只会"说话"，还能调用你定义的函数。这是构建 AI Agent 的基础能力。

🔧 Tool Use 的原理：你告诉 AI "我有这些工具可以用"，AI 根据用户请求决定要不要调用某个工具、传什么参数，然后你执行这个函数，把结果返回给 AI，AI 再给出最终回答。
定义工具 — 用 tool() 函数
import { generateText, tool } from 'ai'
import { openai } from '@ai-sdk/openai'
import { z } from 'zod' // AI SDK 用 zod 定义参数类型

const { text } = await generateText({
  model: openai('gpt-4o-mini'),
  prompt: '北京今天天气怎么样？',
  tools: {
    getWeather: tool({
      description: '获取指定城市的天气信息',
      parameters: z.object({
        city: z.string().describe('城市名称'),
        unit: z.enum(['celsius', 'fahrenheit'])
          .default('celsius'),
      }),
      execute: async ({ city, unit }) => {
        // 这里可以真的调用天气 API
        return { temperature: 22, condition: '晴', city }
      },
    }),
  },
})

console.log(text)
// "北京今天天气晴朗，气温 22°C，适合外出。"
多工具 + maxSteps（自动多步调用）
const result = await generateText({
  model: openai('gpt-4o'),
  prompt: '查询北京和上海的天气，然后比较哪个更热',
  tools: {
    getWeather: tool({/* ... */}),
    compareWeather: tool({/* ... */}),
  },
  maxSteps: 5, // AI 可以自动连续调用多次工具
})

// 查看 AI 的推理步骤
for (const step of result.steps) {
  console.log('工具调用:', step.toolCalls)
  console.log('工具结果:', step.toolResults)
}
toolChoice — 控制工具使用策略
generateText({
  model: openai('gpt-4o-mini'),
  tools: { /* ... */ },
  
  // auto: AI 自己决定要不要用工具（默认）
  toolChoice: 'auto',
  
  // required: 强制 AI 必须用工具
  toolChoice: 'required',
  
  // none: 禁止用工具
  toolChoice: 'none',
  
  // 指定必须用某个特定工具
  toolChoice: { type: 'tool', toolName: 'getWeather' },
})
💡 需要先 npm install zod。Zod 提供了运行时类型校验，AI SDK 用它来确保 AI 传来的参数类型正确，同时也会把 schema 发给模型告知参数结构。

CHAPTER 06
generateObject — 结构化输出

让 AI 直接返回 JSON 对象而不是自由文本。告别手动解析，直接拿到类型安全的数据。

📦 为什么需要结构化输出？
如果让 AI 返回文本，你还需要自己解析。用 generateObject，AI SDK 保证返回的数据严格符合你定义的 Zod schema，省去了大量解析代码。
基础用法
import { generateObject } from 'ai'
import { openai } from '@ai-sdk/openai'
import { z } from 'zod'

const { object } = await generateObject({
  model: openai('gpt-4o-mini'),
  schema: z.object({
    name: z.string(),
    age: z.number(),
    skills: z.array(z.string()),
    isStudent: z.boolean(),
  }),
  prompt: '生成一个虚构的程序员角色信息',
})

console.log(object)
// { name: '张伟', age: 27, skills: ['TypeScript', 'React'], isStudent: false }
// ✅ 完全类型安全，无需 JSON.parse！
实际应用：从文本提取结构化数据
const email = `
  发件人: 王经理
  时间: 明天下午3点
  地点: 会议室B
  内容: 季度复盘会议，请准备Q3数据报告
`

const { object: meeting } = await generateObject({
  model: openai('gpt-4o-mini'),
  schema: z.object({
    title: z.string(),
    organizer: z.string(),
    location: z.string(),
    time: z.string(),
    preparation: z.array(z.string()),
  }),
  prompt: `从以下邮件中提取会议信息：\n${email}`,
})

console.log(meeting.title)       // "季度复盘会议"
console.log(meeting.organizer)    // "王经理"
console.log(meeting.preparation)  // ["准备Q3数据报告"]
output 模式选择
// 模式1: object（默认）—— 返回单个对象
generateObject({ schema: z.object({...}), ... })

// 模式2: array —— 返回对象数组（批量生成）
const { object: products } = await generateObject({
  model: openai('gpt-4o-mini'),
  output: 'array',
  schema: z.object({
    name: z.string(),
    price: z.number(),
  }),
  prompt: '生成5个虚构的软件产品及其价格',
})
// products: [{ name: 'CloudSync Pro', price: 99 }, ...]

// 模式3: enum —— 让 AI 做选择题
const { object: sentiment } = await generateObject({
  model: openai('gpt-4o-mini'),
  output: 'enum',
  enum: ['positive', 'negative', 'neutral'],
  prompt: '今天发布新功能，用户反馈很好！',
})
// sentiment: 'positive'
streamObject — 流式结构化输出（大列表神器）
import { streamObject } from 'ai'

const { partialObjectStream } = streamObject({
  model: openai('gpt-4o-mini'),
  schema: z.object({
    items: z.array(z.object({
      title: z.string(),
      desc: z.string(),
    }))
  }),
  prompt: '列出10个学习 TypeScript 的建议',
})

// 随着生成进行，partial 对象逐渐填充
for await (const partial of partialObjectStream) {
  console.log(partial.items?.length) // 逐渐从 0 增加到 10
}
