CHAPTER 01 Embedding
Embedding 与语义搜索

Embedding 把文本转换成一串数字（向量），语义相近的文本向量也相近。这是构建语义搜索、RAG（检索增强生成）的核心技术。

🧮
什么是向量？
"猫" 和 "狗" 转成向量后距离很近，"猫" 和 "核武器" 距离很远。embedMany 把文本变成向量，cosineSimilarity 计算两个向量有多相似（-1 到 1，越接近 1 越相似）。
语义搜索完整流程
用户输入文本 ──▶ embed() 生成向量 ──▶ 计算 cosineSimilarity
↕ 离线阶段
知识库文档 ──▶ embedMany() 批量向量化 ──▶ 存入向量数据库
安装依赖
# OpenAI 的 text-embedding-3-small 是最常用的 embedding 模型
npm install ai @ai-sdk/openai
embed() — 单条文本向量化
import { embed } from 'ai'
import { openai } from '@ai-sdk/openai'

const { embedding } = await embed({
  model: openai.embedding('text-embedding-3-small'),
  value: '今天天气真不错',
})

console.log(embedding)
// [0.021, -0.083, 0.412, ...] 长度 1536 的数组
console.log(embedding.length) // 1536
embedMany() — 批量向量化（构建知识库）
import { embedMany, cosineSimilarity } from 'ai'
import { openai } from '@ai-sdk/openai'

// 知识库文档
const docs = [
  '退款政策：购买后7天内可申请全额退款',
  '配送说明：全国包邮，3-5个工作日送达',
  '会员权益：黄金会员享受9折优惠',
  '售后服务：联系客服 400-000-0000',
]

// 批量生成向量（一次 API 请求，更高效）
const { embeddings } = await embedMany({
  model: openai.embedding('text-embedding-3-small'),
  values: docs,
})

// 将文档与向量配对存储
const vectorDB = docs.map((doc, i) => ({
  text: doc,
  vector: embeddings[i],
}))

// ── 查询阶段 ──
const query = '我想退货'
const { embedding: queryVec } = await embed({
  model: openai.embedding('text-embedding-3-small'),
  value: query,
})

// 找出最相似的文档
const results = vectorDB
  .map(item => ({
    text: item.text,
    score: cosineSimilarity(queryVec, item.vector),
  }))
  .sort((a, b) => b.score - a.score)

console.log(results[0])
// { text: '退款政策：购买后7天内...', score: 0.89 }
完整 RAG 示例 — 检索增强生成

把语义搜索和文本生成结合，让 AI 基于你的知识库回答问题：

1
检索相关文档
用用户的问题查询向量数据库，取出最相关的 top-k 文档。
2
构建上下文 Prompt
将检索到的文档拼入 system prompt，告诉 AI "基于以下内容回答"。
3
生成回答
调用 generateText / streamText，AI 基于检索内容给出准确答案。
async function ragQuery(userQuestion: string) {
  // Step 1: 向量化用户问题
  const { embedding } = await embed({
    model: openai.embedding('text-embedding-3-small'),
    value: userQuestion,
  })

  // Step 2: 从向量库检索 top 3 相关文档
  const topDocs = vectorDB
    .map(item => ({
      ...item,
      score: cosineSimilarity(embedding, item.vector),
    }))
    .sort((a, b) => b.score - a.score)
    .slice(0, 3)
    .filter(d => d.score > 0.5) // 过滤相关性太低的

  // Step 3: 构建带上下文的 prompt
  const context = topDocs.map(d => d.text).join('\n')

  const { text } = await generateText({
    model: openai('gpt-4o-mini'),
    system: `你是客服助手。请基于以下知识库内容回答问题，
如果知识库中没有相关信息，请说"这个问题我需要转人工处理"。

知识库：
${context}`,
    prompt: userQuestion,
  })

  return { answer: text, sources: topDocs }
}

// 使用
const result = await ragQuery('我能退货吗？')
console.log(result.answer)
// "根据退款政策，购买后7天内可申请全额退款..."
💡
生产环境建议：上面用内存数组模拟了向量库。实际生产中应用真正的向量数据库，推荐 pgvector（PostgreSQL 插件）、Pinecone（托管服务）或 Qdrant（自托管）。

CHAPTER 02 多模态
多模态输入

现代 AI 模型不只能处理文字，还能理解图片、PDF、文件。AI SDK 用统一的 messages 格式处理所有类型的输入内容。

🖼️
AI SDK 的多模态输入通过在 messages 的 content 字段传入数组来实现。每个元素是一个 content part，类型可以是 text、image 或 file。
图片输入 — URL 方式
import { generateText } from 'ai'
import { openai } from '@ai-sdk/openai'

const { text } = await generateText({
  model: openai('gpt-4o'), // 需要支持视觉的模型
  messages: [
    {
      role: 'user',
      content: [
        {
          type: 'image',
          image: new URL('https://example.com/chart.png'),
        },
        {
          type: 'text',
          text: '这张图表说明了什么趋势？用中文回答。',
        },
      ],
    },
  ],
})

console.log(text)

实战：Next.js 图片上传分析 API
app/api/analyze-image/route.ts
import { generateText } from 'ai'
import { openai } from '@ai-sdk/openai'

export async function POST(req: Request) {
  const formData = await req.formData()
  const file = formData.get('image') as File
  const prompt = formData.get('prompt') as string

  const buffer = Buffer.from(await file.arrayBuffer())

  const { text } = await generateText({
    model: openai('gpt-4o'),
    messages: [{
      role: 'user',
      content: [
        { type: 'image', image: buffer, mimeType: file.type },
        { type: 'text', text: prompt || '描述这张图片' },
      ],
    }],
  })

  return Response.json({ result: text })
}
app/page.tsx (前端)
'use client'
import { useState } from 'react'

export default function Page() {
  const [result, setResult] = useState('')

  const handleUpload = async (e: React.ChangeEvent<HTMLInputElement>) => {
    const file = e.target.files?.[0]
    if (!file) return

    const fd = new FormData()
    fd.append('image', file)
    fd.append('prompt', '详细描述这张图片的内容')

    const res = await fetch('/api/analyze-image', {
      method: 'POST', body: fd,
    })
    const data = await res.json()
    setResult(data.result)
  }

  return (
    <div>
      <input type="file" accept="image/*" onChange={handleUpload} />
      {result && <p>{result}</p>}
    </div>
  )
}

CHAPTER 03 Middleware
Provider Middleware

生产环境中你需要日志、缓存、限速、成本控制。Provider Middleware 让你在不改变业务代码的情况下，给任意模型调用加上这些能力。

⚙️
Middleware 原理：wrapLanguageModel() 用装饰器模式包装一个模型对象，拦截每次调用的参数（transformParams）和结果（wrapGenerate / wrapStream），在不侵入业务代码的前提下注入横切逻辑。
Middleware 拦截流程
generateText() ──▶ middleware.wrapGenerate ──▶ transformParams ──▶ model API
──▶ 处理结果 / 记录日志 ◀── 返回结果
基础结构 — wrapLanguageModel
import { wrapLanguageModel } from 'ai'
import { openai } from '@ai-sdk/openai'

const myModel = wrapLanguageModel({
  model: openai('gpt-4o-mini'),
  middleware: {
    // 拦截并修改请求参数
    transformParams: async ({ params }) => {
      return params // 直接返回不修改
    },

    // 拦截非流式调用（generateText）
    wrapGenerate: async ({ doGenerate, params }) => {
      const result = await doGenerate()
      return result
    },

    // 拦截流式调用（streamText）
    wrapStream: async ({ doStream, params }) => {
      const result = await doStream()
      return result
    },
  },
})
实战 1 — 日志 Middleware（记录 token 用量）
function createLoggingMiddleware(label: string) {
  return {
    wrapGenerate: async ({ doGenerate, params }) => {
      const start = Date.now()

      try {
        const result = await doGenerate()
        const ms = Date.now() - start

        console.log({
          label,
          model: params.model,
          promptTokens: result.usage?.promptTokens,
          completionTokens: result.usage?.completionTokens,
          latencyMs: ms,
          finishReason: result.finishReason,
        })

        return result
      } catch (err) {
        console.error({ label, error: err, latencyMs: Date.now() - start })
        throw err
      }
    },
  }
}

// 使用
const loggedModel = wrapLanguageModel({
  model: openai('gpt-4o-mini'),
  middleware: createLoggingMiddleware('chat-api'),
})

// 每次调用自动打印：
// { label: 'chat-api', promptTokens: 45, completionTokens: 123, latencyMs: 892 }
实战 2 — 缓存 Middleware（相同请求不重复调用）
import { createHash } from 'crypto'

// 简单内存缓存（生产环境可换成 Redis）
const cache = new Map<string, any>()

const cachingMiddleware = {
  wrapGenerate: async ({ doGenerate, params }) => {
    // 用请求参数生成缓存 key
    const key = createHash('sha256')
      .update(JSON.stringify(params))
      .digest('hex')

    if (cache.has(key)) {
      console.log('[cache HIT]', key.slice(0, 8))
      return cache.get(key)!
    }

    console.log('[cache MISS]', key.slice(0, 8))
    const result = await doGenerate()
    cache.set(key, result)
    return result
  },
}

const cachedModel = wrapLanguageModel({
  model: openai('gpt-4o-mini'),
  middleware: cachingMiddleware,
})

// 第一次调用：发 API 请求
await generateText({ model: cachedModel, prompt: '天空是什么颜色' })
// 第二次完全相同的调用：直接返回缓存，0 延迟，0 费用！
await generateText({ model: cachedModel, prompt: '天空是什么颜色' })
实战 3 — 限速 Middleware（防止 API 超限）
function createRateLimiter(maxRPM: number) {
  const queue: number[] = [] // 请求时间戳队列

  return {
    wrapGenerate: async ({ doGenerate }) => {
      const now = Date.now()
      const windowStart = now - 60_000 // 1 分钟窗口

      // 清理过期的请求记录
      while (queue.length && queue[0] < windowStart) queue.shift()

      if (queue.length >= maxRPM) {
        const waitMs = queue[0] + 60_000 - now
        console.log(`[rate limit] 等待 ${waitMs}ms`)
        await new Promise(r => setTimeout(r, waitMs))
      }

      queue.push(Date.now())
      return doGenerate()
    },
  }
}

const rateLimitedModel = wrapLanguageModel({
  model: openai('gpt-4o-mini'),
  middleware: createRateLimiter(50), // 最多 50 RPM
})
实战 4 — 叠加多个 Middleware
import { wrapLanguageModel } from 'ai'

// 多个 middleware 可以嵌套叠加
const productionModel = wrapLanguageModel({
  model: wrapLanguageModel({
    model: wrapLanguageModel({
      model: openai('gpt-4o-mini'),
      middleware: cachingMiddleware,        // 最内层：先查缓存
    }),
    middleware: createRateLimiter(50),    // 中间层：限速
  }),
  middleware: createLoggingMiddleware('prod'), // 最外层：记录日志
})

// 执行顺序：日志进入 → 限速检查 → 缓存查找 → API 调用 → 日志输出
const { text } = await generateText({
  model: productionModel,
  prompt: '你好',
})
实战 5 — transformParams：注入系统提示词

在所有请求里强制注入 system prompt，不需要每次手写：

const systemInjector = {
  transformParams: async ({ params }) => ({
    ...params,
    prompt: {
      ...params.prompt,
      // 在现有 system 前面插入公司级别的约束
      system: `你是 ACME 公司的 AI 助手。
回答要专业、简洁，不要提及竞争对手。
用户语言：中文。

${params.prompt.system || ''}`,
    },
  }),
}

const brandedModel = wrapLanguageModel({
  model: openai('gpt-4o-mini'),
  middleware: systemInjector,
})

// 调用时不需要写 system，自动注入
const { text } = await generateText({
  model: brandedModel,
  prompt: '介绍一下你自己',
})
💡
最佳实践：把 productionModel 定义在单独的 lib/ai.ts 文件中，全项目共用这一个实例。这样所有调用都自动享有日志、缓存、限速，而业务代码完全不需要感知。
